# Project-school-21

Репозиторий с артефактами бизнес- и системного анализа по учебному проекту School 21.  
Содержит документацию, диаграммы и модели, описывающие предметную область, требования, бизнес-процессы и пользовательские сценарии системы.

---

## Цель проекта
Проект предназначен для практики навыков бизнес- и системного анализа:  
- выявление требований  
- моделирование бизнес-процессов  
- построение доменной модели  
- описание пользовательских историй  
- разработка диаграмм и артефактов анализа

---

## Структура репозитория

- BSA00_Decomposition — декомпозиция системы и её функций, определение границ системы (scope), верхнеуровневые компоненты

- BSA01_Stakeholders — стейкхолдеры проекта, роли, ожидания, карта интересов

- BSA02_Requirements — функциональные и нефункциональные требования, критерии приемки, приоритизация

- BSA03_HowToRequirements — подходы и правила описания требований, используемые шаблоны и примеры

- BSA04_Domains — доменная модель системы: сущности, атрибуты, связи, словарь терминов

- BSA05_Diagrams — аналитические диаграммы (контекстные, use case, sequence, flow и другие)

- BSA06_BPMN — BPMN-модели бизнес-процессов (AS-IS и TO-BE)

- BSA07_UserStory — пользовательские истории (User Stories), критерии приемки, тестовые сценарии и протоколы согласования

---

## Основные артефакты

В рамках проекта разрабатываются следующие аналитические материалы:

- описание требований
- пользовательские истории (User Stories)
- критерии приемки (Acceptance Criteria)
- тестовые сценарии
- BPMN-модели бизнес-процессов
- диаграммы взаимодействия
- доменная модель системы
- словарь терминов

---

## Что посмотреть в первую очередь

Если вы впервые знакомитесь с проектом:

1. BSA02_Requirements — чтобы понять функциональные возможности системы  
2. BSA04_Domains / BSA05_Diagrams — чтобы увидеть модель системы и её логику  
3. BSA06_BPMN — чтобы понять бизнес-процессы и сценарии взаимодействия  
4. BSA07_UserStory — чтобы увидеть пользовательские сценарии и критерии приемки

---

## Используемые методы и подходы

В проекте применяются следующие практики анализа:

- User Stories  
- Acceptance Criteria  
- INVEST-принципы для пользовательских историй  
- 3C-подход (Card, Conversation, Confirmation)  
- BPMN для моделирования процессов  
- Domain Modeling для описания предметной области
