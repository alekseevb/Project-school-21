# Project-school-21

Репозиторий с артефактами бизнес- и системного анализа по учебному проекту School 21.  
Содержит документацию, диаграммы и модели, описывающие предметную область, требования, бизнес-процессы и пользовательские сценарии системы.

---

## Цель проекта
Проект предназначен для практики навыков бизнес- и системного анализа:  
- выявление требований  
- моделирование бизнес-процессов  
- построение доменной модели  
- описание пользовательских историй  
- разработка диаграмм и артефактов анализа

---

Учебные задачи проекта

В рамках проекта выполняется анализ и моделирование двух предметных областей.

## Задача 1 — Система онлайн-записи в барбершоп

Цель — спроектировать систему, обеспечивающую онлайн-запись клиентов на услуги барбершопа.

Система должна позволять клиентам записываться на услуги, выбирать мастера и свободное время, получать автоматические уведомления о записи и оставлять отзывы после посещения. Менеджер управляет расписанием мастеров, подтверждает оказанные услуги, ведет коммуникацию с клиентами и формирует отчеты. Мастера могут просматривать свое расписание и отзывы клиентов.

##### Для всех артефактов, связанных с этим проектом, используется префикс BARB.

---

## Задача 2 — Система управления доставкой заказов

Цель — спроектировать информационную систему для стартапа службы доставки, позволяющую управлять заказами и работой курьеров.

Заказы поступают в систему и обрабатываются оператором. Курьеры через мобильное приложение могут просматривать доступные заказы, бронировать их, забирать из точки выдачи и доставлять клиенту, фиксируя статус выполнения. Диспетчер контролирует выполнение заказов и при необходимости переназначает их. Информация о заказах и доставках передается в бухгалтерскую систему для расчета оплаты.

##### Для всех артефактов, связанных с этим проектом, используется префикс DLVY.

## Структура репозитория

- BSA00_Decomposition — декомпозиция системы и её функций, определение границ системы (scope), верхнеуровневые компоненты

- BSA01_Stakeholders — стейкхолдеры проекта, роли, ожидания, карта интересов

- BSA02_Requirements — функциональные и нефункциональные требования, критерии приемки, приоритизация

- BSA03_HowToRequirements — подходы и правила описания требований, используемые шаблоны и примеры

- BSA04_Domains — доменная модель системы: сущности, атрибуты, связи, словарь терминов

- BSA05_Diagrams — аналитические диаграммы (контекстные, use case, sequence, flow и другие)

- BSA06_BPMN — BPMN-модели бизнес-процессов (AS-IS и TO-BE)

- BSA07_UserStory — пользовательские истории (User Stories), критерии приемки, тестовые сценарии и протоколы согласования
